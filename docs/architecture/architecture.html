Backend is an npm-workspace monorepo with a strict dependency hierarchy. Arrows point downward: a package may only import from layers below it. contracts is the shared vocabulary, core is the kernel, the four domains sit on top, ai stays deliberately isolated, and server is the only composition root that wires everything together.

Four representative paths through the system — from a metric being scraped, to an AI chat, to a gated remediation, to an OTLP trace arriving from the kernel.

📈 Monitoring cycle

Scheduler · every 5 minutes (metrics every 60s)
Collect stats
Scheduler pulls CPU/mem/net for every running container via the Portainer client
Dual-write metrics
Each sample → PostgreSQL metrics + TimescaleDB hypertable
Detect anomalies
ai-intelligence runs robust-MAD / isolation-forest detectors, scores against baselines
Correlate & persist
Group into incidents, write insights + investigations
Broadcast & notify
insight:new over /monitoring socket; webhooks + notifiers fire

🧠 LLM chat assistant

Socket.IO /llm namespace · streamed
User message
Client emits chat:message (authenticated socket) with prompt + context
Prompt guard
3 layers: 25+ regexes → heuristic scoring → output sanitization
LLM call (+ tools)
POST to LLM_API_URL/v1/chat/completions; MCP tools can read logs mid-chat
Stream back
chat:response token-by-token, then chat:complete
Trace it
Call recorded in llm_traces; user feedback to llm_feedback

Gated by admin role + approval

Execute

Act via Portainer
