Five acts

Week 1 · Jan 31 – Feb 5

Act I — Scaffolding

Born as v2 from day one. The first commits stand up Fastify + React 19 + PostgreSQL.

  • Home dashboard, Workload Explorer, Fleet Overview, Container Health pages
  • Test infrastructure added in commit #4
  • Catppuccin theme pack lands within the first week
  • Custom error boundary, login rate limiting, type-checked frontend build
Feb 6 – Feb 15 · the 717-commit week

Act II — Feature avalanche

The single most productive week of the project. Almost every "headline" feature lands here.

  • SQLite → PostgreSQL migration (Phase 3, AppDb abstraction)
  • Batch anomaly detection + AI insight pipeline (#544–#549)
  • Discord & Telegram notification channels
  • OIDC group-to-role mapping (first cut)
  • UI tiers 1–3: spotlight glow, 3D tilt, motion tokens, page transitions
  • "Neural Search" command palette (macOS Spotlight aesthetic)
  • Harbor Registry vulnerability management — Phase 1
Feb 16 – Feb 28 · v2.0.0 released Feb 23

Act III — Modular kernel

The codebase is rebuilt as an npm-workspace monorepo. Nine "Phase 3.x" extractions.

  • @dashboard/core kernel (config / utils / db / models / plugins / tracing)
  • @dashboard/contracts, ai-intelligence, infrastructure, operations, security
  • @dashboard/server as the composition root, @dashboard/ai
  • Typed event bus, ESLint boundary rules, per-module CLAUDE.md
  • TypeScript project references + per-package CI builds
  • pcap sidecar replaces exec-based capture
Mar 1 – Mar 30

Act IV — Hardening & reach

Pace cools; focus shifts to integrations, guardrails, and config consolidation.

  • Kubernetes endpoint support — pods, deployments, services, namespaces
  • Prometheus + Grafana monitoring overlay
  • Frontend route guards + global error-handling strategy
  • Settings DB becomes primary config source (env-var surface shrinks)
  • Dynamic monitoring scheduler — live reload without restart
  • Shared ConfirmDialog replaces window.confirm; a11y for image treemap
Mar 31 – May 3 · ~5 weeks

Act V — The quiet

Just 4 commits in 34 days. Two LLM fixes (provider switching, LiteLLM connection state). A planning pause before the next push.

  • Apr 17: subscribe to LLM socket events for live connection state
  • Apr 26: /api/llm/query respects custom endpoint settings
May 4 – May 22

Act VI — Observability platform

314 commits in three weeks. The dashboard graduates from "monitor containers" to "understand the runtime".

  • Beyla / eBPF deployment with kernel pre-flight + Edge Agent tunnel checks
  • OTLP trace ingestion, RED query service, sampler, retention job
  • Trace Explorer ↔ logs correlation, container Calls tab, RPC topology overlay
  • Trace-driven latency & error-rate anomaly detection; LLM latency breakdown
  • Canonical EmptyState + Skeleton primitive set across the UI
  • Dedup-engine telemetry + research-question decisions
  • OIDC overhaul: auto redirect URI, group prefix stripping, discovered-groups dropdown
